### Логика обновления (`common/sheets.py`)
- Читает существующие данные листа
- Мержит по `Номер отправления` (обновляет статусы, добавляет новые)
- Сортирует по дате по возрастанию (свежие снизу)
- Записывает обратно через `worksheet.resize()` + `worksheet.update()`
- Время обновления — UTC+3 (Москва)

### Источники данных Ozon API
- **FBS**: `POST /v3/posting/fbs/list` с `financial_data: true`
- **FBO кластеры**: `POST /v2/posting/fbo/list` с `financial_data: true`
- **FBO цены**: `POST /v1/report/postings/create` → CSV отчёт (поле "Оплачено покупателем")
- **Лимит Reports API**: максимум 3 месяца на один запрос

### Историческая загрузка
Данные загружены за периоды (по 3 месяца):
- 2025-06-01 → 2025-09-01
- 2025-09-01 → 2025-12-01
- 2025-12-01 → 2026-03-01
- 2026-03-01 → 2026-06-01

Для загрузки следующего периода: изменить `DATE_FROM` / `DATE_TO` в `ozon/load_history.py` (BM) или `ozon/load_history_cf.py` (CF) и запустить воркфлоу вручную.

### Важно: порядок ключей Ozon
- `Client-Id` — числовое значение (например: `1015676`)
- `Api-Key` — UUID (например: `39cdc157-75ea-423c-a257-48bf919da2fd`)
- Если перепутать — ошибка `"Client-Id header value should be positive integer"`

### Известные проблемы и решения
- **Лимит 10М ячеек Google Sheets**: решено через `worksheet.resize()` перед записью вместо `worksheet.clear()`
- **FBO v3 возвращает 0 заказов**: используем v2 endpoint
- **`customer_price` нет в v2 API**: берём из CSV отчёта
- **Потеря данных при ошибке**: `clear()` заменён на `resize()` — при падении `update()` данные не стираются
- **Reports API лимит 3 месяца**: историю грузим по 3 месяца за раз
- **Сортировка**: свежие заказы снизу (по возрастанию даты)
